Minutes of Meeting: Security Audit Sync – Project AegisDate: April 22, 2026Time: 10:00 AM – 10:25 AMLocation: Conference Room B / ZoomChair: Alex (Security Lead)Note Taker: Jordan (Product Manager)Attendees: Alex, Jordan, Priya (Senior Dev)1. Purpose of MeetingThe session was convened to review the vulnerabilities identified in the recent penetration test of the User Authentication Module and to determine a remediation path before the Q3 Beta release.2. Key Discussion PointsJWT Vulnerability: Alex highlighted that the current JSON Web Token (JWT) implementation uses a "None" algorithm fallback, which could allow an attacker to bypass authentication by modifying the token header.Token Storage: Priya noted that tokens are currently stored in localStorage, making them susceptible to Cross-Site Scripting (XSS) attacks. She suggested migrating to HttpOnly cookies.Session Expiration: Jordan raised concerns that the current 24-hour session window is too long for a financial application. A shorter window of 30 minutes was proposed but met with concerns regarding user experience friction.3. Decisions ReachedProtocol Change: The team agreed to immediately disable the "None" algorithm in the JWT library and enforce RS256 signing.Storage Migration: It was officially decided to move all session tokens to HttpOnly and Secure cookies by the next sprint.Session Timeout: A compromise was reached to set a 60-minute absolute timeout with a 15-minute sliding window.4.
